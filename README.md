# PetMatch-backend

## Конфигурация

Для локального запуска использовать зависимые сервисы из [Docker Compose](./docker-compose.yml), а также указывать
активный Spring профиль "local" (_SPRING_PROFILES_ACTIVE_ или в IDE через _Edit configurations..._). Для локального
запуска всех контейнеров использовать значения из [.env](./.env). Docker
Compose сам увидит файл и прочитает значения из него.

Приложение имеет два профиля:

1. [default](./src/main/resources/application.yaml) для запуска на сервере, в котором конфигурационные значения задаются
   из переменных окружения
2. [local](./src/main/resources/application-local.yaml) для запуска на локальном машине, в котором значения уже
   настроены на локальное использование сервисов из [Docker Compose](./docker-compose.yml).

## Инфраструктура

Помимо самого Backend приложения, рядом разворачиваются следующие сервисы:

1. Postgres в качестве базы данных приложения (сервис **petmatch-postgres**) и экспортер метрик к нему (сервис *
   *petmatch-postgres-metrics-exporter**)
2. Prometheus для хранения метрик приложения (сервис **prometheus**)
3. Loki для хранения логов приложения (сервис **loki**)
4. Tempo для хранения трейсов приложения (сервис **tempo**)
5. Alloy агент для сбора данных с приложения (сервис **alloy**)
6. Grafana для визуализации метрик, логов и трейсов (сервис **grafana**)
7. Redis для кеширования данных приложения (сервис **petmatch-redis**) и экспортер метрик к нему (сервис *
   *redis-exporter**)
8. MinIO для хранения файлов приложения (сервис **petmatch-minio**)

Локально доступны сервисы

1. **Prometheus UI** по адресу localhost:9090 для отслеживания доступности сервисов
2. **Grafana UI** по адресу localhost:3000 для просмотра метрик, логов и трейсов

## Указания

### Авторизация

Для запуска сервиса требуются 2 поля окружения: JWT_SECRET_ACCESS и JWT_SECRET_REFRESH. Сделано для безопасности, ибо
переменные окружения будут ограничиваться безопасностью ОС. Можно настроить так, что переменные окружения будет видеть
только root пользователь.

Примеры создания:
export JWT_SECRET_ACCESS=$(openssl rand -hex 32)
export JWT_SECRET_REFRESH=$(openssl rand -base64 32)
